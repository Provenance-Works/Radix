# Memory Module: Overview

Version: 0.1.0-draft
Last updated: 2026-03-07

## 1. モジュールの目的 (Purpose)

Memoryモジュールは、システム上で動的に確保されるメモリブロックやバッファへの低レベルアクセスを提供します。
FFIはプロジェクト全体で禁止されているため、直接的なC言語の mmap や malloc 呼び出し、および独自仕様のガベージコレクションファイナライザへのフックは一切使用しません。
代わりに、Lean 4のコアが提供する堅牢な ByteArray 等を基盤とし、ゼロコストかつパニックフリーな純粋Lean4環境での動的メモリの読み書きを型レベルで安全に抽象化します。

## 2. アーキテクチャ構成 (Architecture Structure)

Memoryモジュールは、以下のサブコンポーネントから構成されます。

### 2.1 トップレベル (Memory)
直接メモリアドレスの扱いや、メモリ確保解放に関する操作。

- Memory.Buffer: Lean 4 ネイティブの ByteArray に対して、境界外アクセスのパニックを型レベル（証明）で防ぐビューを提供します。
- Memory.Ptr: コンパイラスコープ内で最適化される範囲において、より高レベルな安全ポインタ等価の動作をシミュレートする型。

## 3. 依存関係 (Dependencies)

- **依存先**: Word (アドレス指定やサイズ計算用の UWord 等)
- **依存元**: Binary (直列化／非直列化のためのメモリ基盤)

## 4. 特記事項 (Key Characteristics)

- 本モジュールの中核は、ゼロコピーおよびGCフレンドリーな純粋Leanによる境界設計です。
- リソース解放漏れやUAFのリスクを伴う手動アンマネージドリソース（手動で確保したC領域のポインタなど）は完全に仕様から除外され、全メモリの生存期間は安全なLeanランタイムに直接委譲されます。
