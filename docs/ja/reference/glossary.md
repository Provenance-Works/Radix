# 用語集

> **対象読者**: すべてのユーザー

| 用語 | 定義 |
|------|-----------|
| **TCB** | 信頼コンピューティング基盤（Trusted Computing Base）。正しさが証明ではなく仮定されるコンポーネントの集合。Radix では: Lean 4 コンパイラ/ランタイム、IO 動作の仮定、デフォルト公理、`System.Assumptions` の `trust_*` 公理宣言。 |
| **Layer 1（システムブリッジ）** | 検証済みロジックを OS 操作に接続する Lean 4 コード。Lean 4 IO API を使用。名前付き信頼仮定を含む。 |
| **Layer 2（検証済み実装）** | 正しさの証明を伴う低レベルプリミティブの純粋 Lean 4 実装。 |
| **Layer 3（検証済み仕様）** | 正しい動作を定義する形式仕様と定理。実行可能コードなし。 |
| **BitVec n** | Mathlib の固定幅ビットベクトル型。すべての Radix 整数型の正準的な仕様レベル表現として使用。 |
| **ラッピング算術** | オーバーフロー時に 2^n で剰余をとる算術。例: `UInt8: 255 + 1 = 0`。 |
| **飽和算術** | オーバーフロー時に型の最小/最大値にクランプする算術。例: `UInt8: 255 + 1 = 255`。 |
| **チェック付き算術** | オーバーフロー時に `none` を返す算術。例: `UInt8: checkedAdd 255 1 = none`。 |
| **オーバーフロー付き算術** | ラッピング結果とブールオーバーフローフラグの両方を返す算術。例: `UInt8: overflowingAdd 255 1 = (0, true)`。 |
| **2の補数** | MSB が符号を表す符号付き整数の標準表現。すべての現代ハードウェアがこれを使用。Radix の `Int8`–`Int64` はこのエンコーディングを使用。 |
| **エンディアン** | バイト順序。ビッグエンディアン: 最上位バイトが先。リトルエンディアン: 最下位バイトが先。 |
| **bswap** | バイトスワップ — マルチバイト整数のバイト順序を反転。退化性: `bswap(bswap(x)) = x`。 |
| **ビットフィールド** | より大きな整数内の連続したビット範囲。複数の値を単一のワードにパックするために使用。 |
| **ByteSlice** | 有効な境界の証明を持つ `ByteArray` への境界チェック付きゼロコピービュー。 |
| **パックド構造体** | 暗黙的なパディングのない、明示的に指定されたメモリレイアウトを持つデータ構造。`Memory.Layout` でモデル化。 |
| **プロビナンス** | ポインタがどのアロケーションから発生したかを追跡するメタデータ。`Memory.Ptr` で使用。 |
| **LEB128** | リトルエンディアンベース128。WebAssembly、DWARF、Protocol Buffers で使用される可変長整数エンコーディング。 |
| **ラウンドトリップ特性** | シリアライゼーションの後にデシリアライゼーションが恒等関数となる特性。すべての Radix エンコード/デコード操作で形式的に証明済み。 |
| **DSL** | ドメイン固有言語（Domain-Specific Language）。Radix ではバイナリワイヤフォーマット記述用の `Binary.Format` 帰納型。 |
| **ワイヤフォーマット** | 送信または保存されるデータの正確なバイトレベル表現。 |
| **リファインメント** | 抽象仕様と具体実装の間の形式的関係で、実装が仕様を正しく実現していることを証明するもの。 |
| **POSIX** | ポータブルオペレーティングシステムインターフェース（IEEE Std 1003.1）。Unix 系オペレーティングシステムの標準 API。 |
| **sorry** | 証明なしに任意の命題を認める Lean 4 公理。リリースされた Radix コードには**絶対に**含まれてはならない。 |
| **PlatformWidth** | 自然数を `n = 32 ∨ n = 64` に制約する型クラス。`UWord`/`IWord` のパラメータ化に使用。 |
| **Tier 1 API** | 事前条件の証明を必要とする証明付き API。ランタイムコストゼロ（証明はコンパイル時に消去）。 |
| **Tier 2 API** | 証明要件のない利便性 API。`Option`、ラッピング、飽和、またはオーバーフローフラグを使用。 |
| **MemoryOrder** | アトミック操作のメモリ順序: `relaxed`、`acquire`、`release`、`acqRel`、`seqCst`。 |
| **AtomicCell** | load/store/CAS/fetch 操作をサポートするアトミックメモリロケーションの抽象モデル。 |
| **GCFree** | コードがガベージコレクション対象のヒープアロケーションを使用しないことを保証する制約。ベアメタル環境に必要。 |
| **LinkerScript** | セクション配置、シンボル、メモリ領域を指定する ELF リンカースクリプトのモデル。 |
